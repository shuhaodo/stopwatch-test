## Key Design Decisions

1. **Single HTML File Architecture**
   - All code (HTML, CSS, JS) in one file
   - Rationale: Easy deployment, meets requirements, appropriate for simple app

2. **High-Resolution Timing**
   - Use `performance.now()` instead of `Date.now()`
   - Rationale: Microsecond precision, monotonic, not affected by system clock

3. **RequestAnimationFrame over SetInterval**
   - Use `requestAnimationFrame` for display updates
   - Rationale: Better performance, pauses when tab inactive, 60fps smooth updates

4. **State Management**
   - Simple JavaScript variables (no framework needed)
   - Clear state transitions (initial ‚Üí running ‚Üí stopped ‚Üí reset)

5. **Button States**
   - Start disabled when running
   - Stop disabled when not running
   - Reset always enabled
   - Rationale: Prevents user errors and provides clear feedback

## Technical Specifications

### Display Format
- Seconds with 2 decimal places (e.g., "12.34")
- Monospace font for readability
- Large, centered display

### Timing Accuracy
- Uses `performance.now()` for high-resolution timestamps
- Accumulates elapsed time across start/stop cycles
- No drift even with rapid button clicks

### Browser Compatibility
- Target: All modern browsers (Chrome, Firefox, Safari, Edge)
